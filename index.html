The thesis

Harness design, not model size

Frontier coding agents (Aider, Claude Code, Goose) assume a model that can hold arbitrary context and reason reliably. smallcode inverts this entirely: minimize context, externalize state, constrain output format, decompose tasks, verify deterministically, and sample best-of-N.

VibeThinker-3B raw single-shot HumanEval pass@1 is roughly 0.45–0.60. The agentic loop lifts the same model to 0.828 pass@1 / 0.906 solve-rate on the full TypeScript HumanEval — without touching the model weights.

The bet: if a 3B-class local model with this harness yields solid coding output, capable coding becomes fully offline and private — no API keys, no telemetry, no cloud.

"If harness design unlocks usable coding from a 3B model, that's a game changer."

  • Full-file edit format — no diff markers a small model mangles
  • Edit-apply safety net — fuzzy-repairs drifted search text, rejects truncated file re-emits, and redirects a flattened path (src.stats.ts→src/stats.ts) onto the real file
  • Deterministic test-oracle early-stop — lock the pass, stop burning budget; CLI reports success only when oracle confirms green
  • Repo-context indexing budgeted off the model's real window (num_ctx − generation reserve), with a per-turn trim guard — the prompt never overflows into an HTTP 400
  • Think-only recovery — when a small model burns its budget reasoning and emits no answer, the next turn is re-prompted to skip thinking and act immediately
  • Throughput watchdog — auto-heals KV-cache decay on long runs
  • Best-of-N + strategy-seeded diversity — closes variance gap
  • GEPA prompt-optimization seam — evolve better prompts automatically
  • Offline skill distillation — mine passing sessions into a seed skill block injected alongside the system prompt

Agent loop

Four structural choices that do the heavy lifting

Each turn: fetch context → build prompt → call model → parse reasoning → parse edits → apply edits → execute tools → run oracle → early-stop or advance.

01
planTask → Goals

Decompose the task into ordered sub-goals (max 8). Each goal is attempted independently — failures stay local and self-correction is scoped.

planner
02
Tool-exec turns

Model emits edits and tool calls. The harness actually executes them — read_file, write_file, run_command, run_tests — and feeds real stdout back. No blind flying.

executor
03
Full-file edit format

Model writes FILE: path + the entire corrected file in a fenced block. No diffs, no SEARCH/REPLACE markers. This single change jumped the suite from 0.40 → 0.96.

edit-apply
04
Test-oracle early-stop

After every turn: bun test runs automatically. Tests green = task solved. Break immediately — lock the pass, prevent regression, stop spending budget.

oracle

Get started

Quick start

  1. Install Ollama + pull the model Download from ollama.com/download then pull VibeThinker-3B
  2. Install dependencies Requires Bun runtime
  3. Start the model server Use the bundled script for long sessions (sets KV-cache flags)
  4. Run a task or benchmark Point at any repo — or run the HumanEval-TS suite directly

Config lives in smallcode.config.json. Default points at http://localhost:11434/v1 with model vibethinker-3b. See LLM Reference for all env vars.

shell
# Pull the model and start Ollama ollama pull weiboai/vibethinker-3b ollama serve # Install dependencies (Bun required) bun install # For sessions >1h — sets KV-cache flags, adds watchdog scripts/ollama-serve.sh # Run the first 30 HumanEval problems, k=3 trials SMALLCODE_HE_LIMIT=30 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts # Run the full in-house capability suite (or pick another: SMALLCODE_SUITE=edit-reliability) bun scripts/run-baseline.ts # Code a task in any repo bun run index.ts run --task "Add input validation to src/api/handler.ts" --repo .

Capabilities

What's inside

🗺
Repo-context indexing

Builds a minimal ContextBundle from a RepoMap of symbols and file chunks, respecting a hard token budget. The model never sees more than it can hold.

⚡
Tiered verification oracle

T0 syntax → T1 tsc → T2 lint → T3 build → T4 existing tests → T5 self-tests. bun test is the gold top tier. Early-stop locks the pass the moment tests go green.

📡
Throughput watchdog

Monitors tok/s every request. Auto-unloads and reloads the Ollama model when throughput decays below ~30% of session baseline — self-heals the KV-cache fragmentation that degrades long runs from 105 → 0.5 tok/s.

🎯
Best-of-N sampling

Oracle-verified BoN wraps the agent loop with N outer samples and a temperature sweep [0.7, 1.0, 1.3]. First solution to go green wins. Closes the variance gap that comes with temp=1.0 sampling.

🧬
GEPA prompt-optimization

Generative Evolutionary Prompt Architect seam is live. AgentConfig.promptSet injects candidate prompts end-to-end; src/improve/gepa/ provides the Pareto-front evolution engine with eval-suite fitness function.

📊
Eval & benchmark harnesses

YAML/JSON task suites with deterministic test graders, LLM rubrics, pass@1/pass@k metrics, and a regression gate for CI. HumanEval-TS runner and multi-model bake-off scripts included.

A local coding agent you can actually let touch your repo.

The harness, not the model size.

smallcode wraps local 3B-7B models (qwen2.5-coder) in a harness that constrains their output and verifies it deterministically — and lets you review each proposed edit before it's written to disk, undo exactly what the agent changed (never your own uncommitted work), and escalate to a bigger local model only when the small one stalls. Fully offline. No cloud APIs, no API keys.

The thesis

Harness design, not model size

Frontier coding agents (Aider, Claude Code, Goose) assume a model that can hold arbitrary context and reason reliably. smallcode inverts this entirely: minimize context, externalize state, constrain output format, decompose tasks, verify deterministically, and sample best-of-N.

We built the harness on VibeThinker-3B — it produced the 0.828 HumanEval-TS pass@1 baseline and revealed a "think-only" reasoning spiral (model burns its generation budget inside <think>, emits no code, 0/10 on some real-lib bugs). We confirmed the HARNESS, not the model, does the work by swapping to qwen2.5-coder:3b — same size, no think-only, 30-60x faster — and rerunning the realrepo suite (12 tasks, n=10): qwen-3b 0.70, qwen-7b 0.88 model-only. The harness then goes further: deterministic operator-mutation repair (default on) brute-forces the wrong-comparison-operator bug class that no model-side lever cracks, lifting qwen-7b realrepo to 0.94 (the mri floor task 0.00 → 1.00 — the repair mutates the pristine pre-model file, recovering the trials where the model mangled the target) with zero regression — the harness, not a bigger model, doing the work. VibeThinker still supported via SMALLCODE_MODEL=vibethinker-3b.

The bet: if a 3B-class local model with this harness yields solid coding output, capable coding becomes fully offline and private — no API keys, no telemetry, no cloud.

"If harness design unlocks usable coding from a 3B model, that's a game changer."

  • Full-file edit format — no diff markers a small model mangles
  • Target-file pinning + size-gated patching — the harness pins the edit target as an undroppable whole-file chunk and deterministically picks the format (≤140 lines → whole file, larger → patch one function) so the model never self-assesses file size
  • Edit-apply safety net — fuzzy-repairs drifted search text, rejects truncated file re-emits, and redirects a flattened path (src.stats.ts→src/stats.ts) onto the real file
  • Deterministic test-oracle early-stop — lock the pass, stop burning budget; CLI reports success only when oracle confirms green
  • Repo-context indexing budgeted off the model's real window (num_ctx − generation reserve), with a per-turn trim guard — the prompt never overflows into an HTTP 400
  • Think-only recovery — when a small model burns its budget reasoning and emits no answer, the next turn is re-prompted to skip thinking and act immediately
  • Throughput watchdog — auto-heals KV-cache decay on long runs
  • Best-of-N + strategy-seeded diversity — closes variance gap
  • Read-after-delete repair — two levers crack the lru-recency ordering floor: SMALLCODE_RAD_HINT (model-side ## STATEMENT ORDER BUG hint makes the model reorder the statements, default on since v1.7.1) and SMALLCODE_STATEMENT_REPAIR (harness-side deterministic hoist, opt-in), each lifting pass@1 0.13 → 1.00 (7b, n=8). RAD_HINT is the first model-side lever to move a genuine realrepo floor (audit modelSolveRate 1.0); a full 22-task A/B confirmed it fires on the read-after-delete pattern only (0 fires across the other 21 tasks), so default-on is inert everywhere else. STATEMENT_REPAIR is a harness rescue
  • Static import-resolution gate — SMALLCODE_IMPORT_GATE (default ON since 2026-07-05, set =0 to disable): kills hallucinated imports (the dogfood std/strings failure) before they cost a turn. After an edit lands, it resolves the imports the edit introduced against package.json deps + node_modules + the filesystem; anything that doesn't resolve is reverted and the model gets a targeted "does not resolve; available deps: …" message — proactive grounding vs the reactive post-test-run "Cannot find module". Promoted after live-fire validation: caught + reverted a real 3B hallucinated slugify import, zero false-fires, with the global-cache resolver fix
  • Final-state regression guard — SMALLCODE_FINAL_STATE_GUARD (default ON since 2026-07-05, set =0 to disable): the "never leave the repo worse than found" guarantee. When a run ends UNSOLVED and the repo is strictly worse than baseline (more red tests), the harness restores every touched file to pristine and deletes brand-new files it created. Eval-neutral (only reverts unsolved runs to the seeded-bug start state), closes the dogfood broken-repo gap the per-turn revert can't cover. Promoted after validation: realrepo subset pooled pass@1 0.90 = baseline (neutral)
  • GEPA prompt-optimization seam — evolve better prompts automatically
  • Offline skill distillation — mine passing sessions into a seed skill block injected alongside the system prompt
  • Real-repo eval layer — 22 tasks on vendored real libraries (lukeed/klona, dequal, mri, clsx, dset, regexparam + a 2-file tinycalc, plus seeded arithmetic/logical operator bugs) with authored bun:test oracles, not only synthetic fixtures; every task integrity-checked by scripts/validate-realrepo-tasks.ts. A hard tier of 8 tasks (2026-07-03) de-saturates the stick — 4 multi-file deep-localization (emitter-off, csv-quote, toposort-cycle, template-trim) + 4 single-file multi-line structural fixes immune to operator-mutation repair (lru-recency, memoize-key, qs-array, semver-prerelease); each has no single obvious file and no single-operator fix. The lru-recency read-after-delete floor (0.13) is since CRACKED to 1.00 by two new levers — SMALLCODE_RAD_HINT (model-side, default on) and SMALLCODE_STATEMENT_REPAIR (harness-side, opt-in)
  • Model-agnostic — same harness, VibeThinker-3B → qwen2.5-coder:3b/7b, realrepo 0.70 / 0.88 model-only (12 tasks, n=10; 7b SIGNIFICANT over 3b — the new dset/regexparam tasks are the CI-significant discriminators). The mri-flags task was a hard 0/10 for 3b, 7b AND 32b — a genuine wrong-operator wall no model-side lever cracked — until deterministic operator-mutation repair (default on) solved it, lifting qwen-7b realrepo to 0.94 overall

Agent loop

Four structural choices that do the heavy lifting

Each turn: fetch context → build prompt → call model → parse reasoning → parse edits → apply edits → execute tools → run oracle → early-stop or advance.

01
planTask → Goals

Decompose the task into ordered sub-goals (max 8). Each goal is attempted independently — failures stay local and self-correction is scoped.

planner
02
Tool-exec turns

Model emits edits and tool calls. The harness actually executes them — read_file, write_file, run_command, run_tests — and feeds real stdout back. No blind flying.

executor
03
Full-file edit format

Model writes FILE: path + the entire corrected file in a fenced block. No diffs, no SEARCH/REPLACE markers. This single change jumped the suite from 0.40 → 0.96.

edit-apply
04
Test-oracle early-stop

After every turn: bun test runs automatically. Tests green = task solved. Break immediately — lock the pass, prevent regression, stop spending budget.

oracle

Get started

Quick start

  1. Install Ollama + pull a model Download from ollama.com/download then pull qwen2.5-coder:3b (recommended)
  2. Install dependencies Requires Bun runtime
  3. Start the model server Use the bundled script for long sessions (sets KV-cache flags)
  4. Run a task or benchmark Point at any repo — or run the HumanEval-TS suite directly

Config lives in smallcode.config.json. Default points at http://localhost:11434/v1 with model qwen2.5-coder:3b (recommended). See LLM Reference for all env vars.

shell
# Pull the recommended model and start Ollama ollama pull qwen2.5-coder:3b # or: ollama pull weiboai/vibethinker-3b # origin baseline ollama serve # Install dependencies (Bun required) bun install # For sessions >1h — sets KV-cache flags, adds watchdog scripts/ollama-serve.sh # Run the first 30 HumanEval problems, k=3 trials SMALLCODE_HE_LIMIT=30 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts # Run the full in-house capability suite (or pick another: SMALLCODE_SUITE=edit-reliability | realrepo) bun scripts/run-baseline.ts # Swap the model per-run (no config edit) — qwen2.5-coder:3b / :7b / vibethinker-3b SMALLCODE_MODEL=qwen2.5-coder:3b SMALLCODE_SUITE=realrepo bun scripts/run-baseline.ts # Trustworthy measurement: n samples/task, pass@k with 95% bootstrap CIs SMALLCODE_SUITE=edit-reliability SMALLCODE_EVAL_N=15 SMALLCODE_REPORT_KS=1,2,3,5 bun scripts/run-baseline.ts # Run-level oracle-verified Best-of-N — reported pass@1 IS the empirical pass@N(any) SMALLCODE_SUITE=edit-reliability SMALLCODE_BEST_OF_N=3 SMALLCODE_EVAL_N=10 bun scripts/run-baseline.ts # Focused A/B on a subset (TASK_FILTER = id substrings), and forensic turn-dump of one hard task SMALLCODE_SUITE=realrepo SMALLCODE_TASK_FILTER=mri,dequal bun scripts/run-baseline.ts FORENSIC_TASK=mri FORENSIC_N=5 SMALLCODE_MODEL=qwen2.5-coder:3b bun scripts/forensic-task.ts # Code a task in any repo (task description is a positional, not a flag) smallcode run "Add input validation to src/api/handler.ts" --repo . # Review what the agent changed, then undo it if you don't like it (safe: dry-run without --yes) smallcode diff --repo . smallcode undo --repo . --yes # Or work interactively — keeps repo+model warm; /add to focus, /diff and /undo inline smallcode chat --repo . # Escalation: start on the cheap local model, climb to a bigger LOCAL one only on the residual. # Single-shot escalate-on-failure (default best-of-n=1): runs 7b; if the test still fails it # reverts ONLY its own edits (your work is safe) and retries with 32b. Needs a git repo + a red test. smallcode fix --escalation qwen2.5-coder:7b,qwen2.5-coder:32b # Or best-of-N: N attempts up the ladder, first oracle-green wins. Needs a CLEAN git tree. # Low-resource? Drop --escalation and run the 3b alone. Bigger box? add 7b / gemma4:12b / 32b. smallcode run "fix the failing test" --best-of-n 3 --escalation qwen2.5-coder:3b,qwen2.5-coder:7b,gemma4:12b

Capabilities

What's inside

🗺
Repo-context indexing

Builds a minimal ContextBundle from a RepoMap of symbols and file chunks, respecting a hard token budget. The model never sees more than it can hold.

⚡
Tiered verification oracle

T0 syntax → T1 tsc → T2 lint → T3 build → T4 existing tests → T5 self-tests. bun test is the gold top tier. Early-stop locks the pass the moment tests go green.

📡
Throughput watchdog

Monitors tok/s every request. Auto-unloads and reloads the Ollama model when throughput decays below ~30% of session baseline — self-heals the KV-cache fragmentation that degrades long runs from 105 → 0.5 tok/s.

🎯
Best-of-N sampling

Oracle-verified BoN wraps the agent loop with N outer samples and a temperature sweep [0.7, 1.0, 1.3]. First solution to go green wins. Measured: edit-reliability pass@1 0.84 → 0.97 at N=3 (significant), spending ~1.0–1.8 attempts/trial — it closes the variance gap from temp=1.0 sampling.

🧬
GEPA prompt-optimization

Generative Evolutionary Prompt Architect is live end-to-end. AgentConfig.promptSet injects candidate prompts; src/improve/gepa/ provides the Pareto-front evolution engine with an eval-suite fitness function and a LLMReflectiveMutator — a strong reflection model rewrites the system prompt from failed-task transcripts. Run it with bun scripts/gepa-run.ts (CI-smokeable via SMALLCODE_GEPA_DRY_RUN=1).

📊
Eval & benchmark harnesses

YAML/JSON task suites with deterministic test graders, LLM rubrics, pass@1/pass@k metrics, and a regression gate for CI. HumanEval-TS runner and multi-model bake-off scripts included.

✅
Review before write

With sandbox.requireApproval on (the default), an interactive terminal shows you each proposed edit and asks y/N before writing it. In a non-interactive run (CI, piped, --json) there's no TTY to answer, so edits apply with a one-time notice — review with smallcode diff, roll back with smallcode undo. Pass --yes to apply without prompting even in a terminal.

↩️
diff / undo, scoped

smallcode diff shows exactly what the agent changed. smallcode undo reverts only the agent's edits — recorded per run — and never touches your own uncommitted work.

💬
Interactive chat

smallcode chat keeps the repo index and model warm across multiple tasks in one session. /add to focus context, /diff and /undo inline.

🪜
Escalate to your hardware

A model ladder (3b → 7b → gemma4:12b) runs the small model first and climbs to a bigger local model only on the residual it can't solve — no cloud fallback required.

🛡️
Won't leave your repo broken

On untested code, a static-confidence check reverts an edit that doesn't even parse and reports what it could verify — a safety signal, not a correctness claim.

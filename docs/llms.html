Module map

Every important file in src/ and scripts/ with a one-line purpose.

src/agent/

FilePurpose
loop.tsCore agent turn loop — planning, model calls, edit application, tool execution, baseline-relative test-oracle early-stop. Captures a TestBaseline before the loop starts; passes it to every runTieredOracle call. Implements stall detection: STALL_LIMIT=2 consecutive turns with the same failureSignature trigger a redraft (suppresses Recent History, injects rotateStrategy hint); capped at MAX_REDRAFTS=2. Think-only recovery: when a turn truncates mid-reasoning (hasReasoning && answer==="" && truncated!==false) it is recorded as a failed turn and answerNowNext is set so the next turn is built with { answerNow: true } (suppress history, demand immediate action) instead of re-sending the identical prompt that just truncated. Empty-generation infra guard: a turn whose provider returned zero tokens (rawContent.trim()==="" && completionTokens===0 — a wedged/disconnected local backend that emits {"response":"","done":false}) is thrown and recorded with the error tag "infra: empty model generation", distinct from think-only (which HAS reasoning), so the eval can exclude the trial instead of forging a false 0.00. Stores turn.failureSignature, turn.diagnostic, turn.redrafted, turn.answerNow; persists state.stallCount, state.redraftCount, state.lastFailureSignature. Entry point: runLoop(state, statePath, deps, getContext)
planner.tsCalls the model to decompose a task into ≤5 ordered sub-goals (down from 8). Falls back to a single goal if the model returns nothing parseable. Optional preSolveReflection (default off): one extra cheap call to restate the problem + key edge case, fed into goal generation (AlphaCodium-lite). Context summary is sanitised to unique file paths only (no line ranges, capped at 8 files) so the model cannot echo path labels back as goals. Post-parse filtering via isActionableGoal(desc) rejects bare path / path-with-line-range echoes, and isInstructionEcho(desc) rejects goals that parrot the planner's OWN instruction prose (failure mode #7 — markers like "action verb", "sub-goal", "numbered list", literal <action verb>/<specific thing> placeholders, or the copied example verb list "Add, Fix, Implement"); when every goal is filtered out the planner falls back to the raw task. maxGoalsForTask(task, uniqueFileCount) enforces the cap (single-file → 3, multi-file → 5). Exact-duplicate descriptions are deduplicated. The planner system prompt also carries an explicit "NEVER restate these instructions as a goal" guard. All three helpers are exported for testing.
prompt.tsbuildSystemPrompt — delegates to config.promptSet ?? defaultPromptSet(); when the resolved PromptSet.skill is non-empty, appends \n\n## SKILL\n<skill> to the returned string; otherwise output is byte-identical to the no-skill baseline. The discipline-rules toggle relocates to prompt-set.ts. buildTurnPrompt(state, context, opts?) — assembles task, goal, turn number, last 2 turns of history (suppressed on opts.redraft), context chunks, a ## Edit Target directive derived from context.targetFile (small file ≤80 lines → "emit the COMPLETE file" via FILE:; large file >80 lines with a known function → "use PATCH:/FUNCTION: to replace ONLY that one function" — the harness, not the model, picks the format), and a Failure (fix THIS) block rendered from state.turns.at(-1)?.diagnostic when present. opts.answerNow emits an ANSWER NOW section (skip thinking, act on the first line) and also suppresses history; it takes precedence over redraft. BuildTurnPromptOpts: { redraft?: boolean; strategyHint?: string; answerNow?: boolean }. fitTurnPromptToWindow(state, context, systemPrompt, hardCap, opts?) — pre-flight guard: rebuilds the prompt while shedding the largest non-pinned repo-context chunk each pass until estimateTokens(system) + estimateTokens(prompt) ≤ hardCap or no droppable chunks remain; returns { turnPrompt, estimatedTokens, droppedChunks }. The pinned target file is never shed. History (failing-test output, failed-edit file) is never trimmed. Pure, deterministic, does not mutate the caller's chunk array.
prompt-set.tsPromptSet { system, planner, reflection, skill? } — groups all three agent prompts so a GEPA candidate can carry a mutated variant. The optional skill?: string slot holds a repo-scoped guidance block (mirrors gskill's SKILL.md); when non-empty, buildSystemPrompt appends a ## SKILL section after the base system text. defaultPromptSet(opts?) assembles the canonical set and accepts opts.skill (default undefined). Single source of truth for the prompt strings.
state.tsCreate, load, save (atomic rename), and mutate AgentState. addTurn, advanceGoal, failGoal, isTerminal, currentGoal.
tools.tsTool implementations (read_file, write_file, run_command, run_tests, think, finish) and the executeTool dispatcher. All paths guarded by resolveSafe.
bestofn.tsselectBestCandidate — generate N completions in parallel, apply each, run verifier, pick winner by verifierScore, revert losers, apply winner.
types.tsTypeScript types: AgentState (+ lastFailureSignature?, stallCount?, redraftCount?, verified? — set true only when the tiered oracle confirmed solved before loop exit), AgentConfig (optional promptSet?: PromptSet for GEPA seam), Goal, TurnRecord (+ failureSignature?, redrafted?, answerNow?, diagnostic?), ToolCall, ToolResult, ToolName.
strategy.tsrotateStrategy(n: number): string — returns the nth strategy hint (0-indexed) from a 3-item cycle: "use a set/dedup explicitly", "step through each provided example by hand first", "handle edge cases (empty, negative, boundary) before the main case". Used by loop.ts to vary the redraft nudge each cycle.
index.tsRe-exports the public agent API including PromptSet, defaultPromptSet.

src/edit/

FilePurpose
parser.tsParse model output into EditBlock[]. Tries PATCH: (function-level) first, then full-file, then search-replace, then JSON. Public: parse(raw).
applier.tsApply EditBlock to disk. Full-file (search='') overwrites the whole file — guarded by truncationReason(original, replace) which rejects likely-truncated re-emits (unbalanced brackets vs a balanced original, >50% line loss, blanking a non-empty file) so a dropped tail never silently corrupts a file. Search-replace tries exact match, then repairBlock fuzzy repair on a miss (records ApplyResult.repair). flattenedPathCandidate(path) + applyBatch redirect a dots-flattened path (src.stats.ts → src/stats.ts) onto the real existing file instead of creating a stray one. patch-function blocks dispatch via a sentinel prefix. Public: applyBatch, truncationReason, flattenedPathCandidate.
patch-function.tsFunction-level edits for large files: parsePatchBlocks, applyPatchBlock (anchors on the function's DEFINITION located by NAME via findDefinitionLines — robust to a small model emitting a drifted signature; the synthetic name "default" anchors on the export default function line; falls back to first-replacement-line match only when name lookup finds nothing; brace-depth scan finds the region end), preserveExport (re-adds a leading export/export default the model dropped — a common slip that un-exports the function and breaks importers), chooseEditFormat (PATCH_LINE_THRESHOLD = 80 lines or >8KB → patch eligible; threshold lowered from 300). The harness — not the model — decides the format deterministically from the pinned target file's size and emits it as the ## Edit Target directive. Fail-safe: ambiguous/not-found anchor → error ApplyResult, no write.
repair.tsFuzzy repair for failed search-replace blocks — strategies: exact → whitespace-normalize → per-line-trim → char-similarity fuzzy (≥0.85). repairBlock(block, content) returns the verbatim recovered search text + strategy/confidence. Wired into applier.ts (no longer dead code): a search/replace that misses exact match retries against the repaired text.
types.tsEditBlock, EditFormat (incl. patch-function), ParseResult, ParseError, ApplyResult, ApplyStatus, ApplyBatchResult, RepairResult.
index.tsRe-exports parse and applyBatch.

src/context/

FilePurpose
walker.tsRecursively walk a repo directory, skip node_modules/.git/.smallcode, collect source files by extension.
extractor.tsExtract CodeSymbols (functions, classes, exports, etc.) from source files (TS compiler for TS/JS, regex for other langs). Includes an anonymous export default function (…) as the symbol "default" — common in real libraries (mri, etc.) where the default export IS the main implementation; without it the PATCH target selector could never aim there.
scorer.tsScore ContextChunks by relevance to a query. Used to prioritize chunks within the token budget.
builder.tsBuild a ContextBundle — collect, score, and truncate chunks to fit the token budget (default 8,000 tokens). Target-file pinning: identifies the edit target — the highest query-scored non-test, non-barrel source file (barrel/re-export files doing export * are skipped, they define no implementation) — and emits it as a single PINNED whole-file chunk that is never windowed (always the complete file, never a ±20-line slice) and never shed by fitTurnPromptToWindow. Sets bundle.targetFile = { path, lineCount, format: "full"|"patch", functionName } so the harness can deterministically size-gate the edit format (≤80 lines → full file, >80 → patch one function). Target function (pickTargetFunction): picks the function CONTAINING the bug — a dominant function (covers ≥50% of the file and ≥2× the next) wins query-independently (the common real-repo shape: a default export that IS the implementation), else an exact query name-match, else body-content relevance. Deliberately does NOT trust a weak name-substring match (which once mis-aimed a 9-line helper toVal over a 100-line parser). File tie-break: when several same-named files export the same symbol (a multi-file repo with decoys), the file the repo's TEST files actually import (resolved from their from "…" specifiers) is promoted over non-imported decoys, so the pin lands on the real entry point. Disabled by SMALLCODE_TARGET_PIN=0.
tokens.tsestimateTokens(text) — fast estimate via Math.ceil(text.length / 4).
types.tsContextChunk, ContextBundle, CodeSymbol, FileMap, RepoMap.

FilePurpose

runner.tsrunSuite(suite, opts) — iterate tasks, call runTask, collect EvalRunResult.
task-runner.tsrunTask(task, opts) — create trial env, run agent loop n times (opts.trialsPerTask = sample count), grade each trial, then report pass@k for a SET of k (opts.reportKs, default [1,2,3,5] ∪ {n}) — n is decoupled from the reported k, the core variance fix (the old code only computed pass@k at k=n, collapsing it to a binary). Pure, seeded → reproducible intervals.

trial-env.tscreateTrialEnv(task, fixturesRoot) — copy fixture into a temp dir, return dir path and cleanup fn.

On exhaustion: verdict:"error" with details.infraError=true, so the run excludes that trial from the rate denominator instead of counting it as a model failure.

Tool contracts

The model emits tool calls as TOOL: name {"arg": "value"} lines. The harness parses them with a line-level regex and executes each in order. think and finish are control-flow only; others are side-effecting.

read_file
TOOL: read_file {"path": "src/foo.ts"}

Read a file relative to repoRoot. Returns file contents (truncated at 8,000 chars with [truncated] marker). Returns (file not found) if missing. Path must stay within repoRoot (traversal check).

  • path (string, required) — relative path to file
  • Output fed back as tool result in next turn's prompt history
write_file
TOOL: write_file {"path": "src/foo.ts", "content": "..."}

Write content to a file. Creates intermediate directories. If requireApproval: true, throws ApprovalRequiredError before writing. Path traversal check enforced. Prefer the FILE: edit format — it is parsed before tool calls and is the intended write mechanism.

  • path (string, required)
  • content (string, required)
run_command
TOOL: run_command {"cmd": "bun tsc --noEmit"}

Run a shell command in repoRoot. Command basename must be in allowedCommands (default: bun, tsc, biome, git). Timeout: 30 seconds. Output truncated at 4,000 chars. Combined stdout+stderr returned.

  • cmd (string, required) — space-separated argv
  • Returns: { success: exitCode===0, output, durationMs }
run_tests
TOOL: run_tests {}

Run bun test (or bun test <pattern>) in repoRoot. Timeout: 60 seconds. "Solved" means zero new failures relative to a pre-loop baseline AND ≥1 passing test. Pre-existing failures captured in the baseline are excluded, so unrelated failing tests in a real repo never block early-stop. On fresh single-file benchmark repos (empty baseline) the behaviour is identical to the original whole-suite-green check. This is the deterministic pass-oracle: when outcome is "solved", the loop breaks immediately.

  • pattern (string, optional) — test file glob/path
  • Returns: { success: exitCode===0 && failCount===0, output, durationMs }
think
TOOL: think {"content": "reasoning here..."}

No-op control tool. The harness records it but takes no action. Allows the model to emit explicit reasoning steps outside <think> tags. Always returns success.

finish
TOOL: finish {"summary": "implemented add()"}

Signal goal completion. The harness calls advanceGoal — marks the current goal done and increments currentGoalIndex. If all goals are done, status becomes done. Note: if the oracle returns "solved" (zero new failures + ≥1 pass vs baseline), the loop early-stops before checking for finish — the baseline-relative early-stop oracle takes precedence.

  • summary (string, optional) — brief description of what was done

Edit format spec

Primary: full-file

Emit FILE: <path> on its own line, then a fenced code block with the complete corrected file. The search field is empty — the applier overwrites the whole file. This is what the system prompt instructs and what VibeThinker-3B reliably emits.

# Full-file format FILE: src/math.ts ```ts export function add(a: number, b: number): number { return a + b; } ``` TOOL: run_tests {} TOOL: finish {"summary": "implemented add()"}

Rules enforced by the parser: path must not be empty, absolute, or contain ... Leading ./ is normalized. Backslashes are converted to forward slashes.

Large files: patch-function

For files past chooseEditFormat's threshold (PATCH_LINE_THRESHOLD = 80 lines or >8KB; lowered from 300), re-emitting the whole file wastes context and risks the small model dropping unchanged lines. The PATCH: format replaces a single function instead. The harness picks this format deterministically from the pinned target file's line count and tells the model via the ## Edit Target directive — the model never self-assesses file size. The anchor is the first line of the replacement body (typically the signature); the applier finds the one matching line, brace-depth-scans to the closing }, and swaps only that region. If the anchor is missing or ambiguous it returns an error ApplyResult (no write) so the loop can fall back to full-file — it never silently mis-applies.

# patch-function format (large files) PATCH: src/foo.ts FUNCTION: parseConfig ```ts export function parseConfig(raw: string): Config { return JSON.parse(raw); } ```

Fallback 1: search-replace

Classic diff-like format. The file path is on the line immediately before the <<<<<<< SEARCH marker (3+ angle brackets, case-insensitive). VibeThinker-3B has trouble with this format; use full-file instead.

src/math.ts <<<<<<< SEARCH function add(a, b) { return a; } ======= function add(a: number, b: number): number { return a + b; } >>>>>>> REPLACE

Fallback 2: JSON

JSON object or array with "type": "edit". Extracted from fences or bare JSON spans. Only activated if the raw output contains "type":"edit".

{"type": "edit", "file": "src/math.ts", "search": "return a;", "replace": "return a + b;"}

Parse priority

Priority Format Trigger Notes
1st patch-function PATCH: marker Checked first, but only matches when a PATCH: block is present. The harness gates this format on the pinned target file's size (>80 lines) via the ## Edit Target directive. Anchored, fail-safe.
2nd Full-file FILE: marker Returns immediately if blocks found and valid. Primary/default format.
3rd Search-replace <<< SEARCH / ===== / >>> REPLACE Fallback. VibeThinker-3B unreliable on this format.
4th JSON "type":"edit" anywhere in output Last resort. Also parses from markdown fences.

smallcode.config.json

Config file searched at smallcode.config.json, then .smallcode.json. Override location with explicit path arg to loadConfig(path).

{ "config": { "provider": { "baseUrl": "http://localhost:11434/v1", // Ollama OpenAI-compat endpoint "apiKey": "none", // unused for local Ollama "timeoutMs": 180000 // 3 min per HTTP call }, "activeModel": "vibethinker-3b", "maxTurns": 15, "bestOfN": 1, "sandbox": { "enabled": true, "requireApproval": false, // false = fully automated "allowedCommands": ["bun", "tsc", "biome", "git"], "networkAccess": false }, "eval": { "suitesDir": "evals/suites", "transcriptsDir": "evals/transcripts", "defaultTrials": 5 } }, "models": [] // optional: custom ModelProfile objects }

How to run

Run the HumanEval-TS benchmark

Fetches problems from the HuggingFace API. Requires Ollama running with VibeThinker-3B.

# First 30 problems, k=3 (reproduces the 0.967 result) SMALLCODE_HE_LIMIT=30 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts # Full set (159 problems, k=3) — expect 8-12 hours SMALLCODE_HE_LIMIT=164 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts 2>&1 | tee /tmp/humaneval-full.log # Re-run timeout-affected problems with fresh Ollama + 20-min cap bun scripts/rerun-timeouts.ts /tmp/humaneval-full.log

Run the in-house capability suite

# Dry-run: validate all reference solutions pass their graders (no agent needed) SMALLCODE_DRY_RUN=1 bun scripts/run-baseline.ts # Live run: k=3 trials per task with the agent SMALLCODE_EVAL_K=3 bun scripts/run-baseline.ts # Faster: k=1, max 3 turns per task SMALLCODE_EVAL_K=1 SMALLCODE_EVAL_MAX_TURNS=3 bun scripts/run-baseline.ts # Select a different suite under evals/suites/ (default: capability) # edit-reliability = real multi-file textkit repo (26 tests, files spanning the size # spectrum) measuring edit-localization reliability, distinct from synthetic suites SMALLCODE_SUITE=edit-reliability bun scripts/run-baseline.ts # realrepo = vendored real libraries (lukeed/klona deep-clone + lukeed/dequal deep-equal, # both MIT, zero-dep) with seeded bugs and authored bun:test oracles — tests real-world # coding (multi-file retrieval, library internals) beyond the synthetic textkit fixtures SMALLCODE_SUITE=realrepo bun scripts/run-baseline.ts

Debug a single task

Runs one task end-to-end, preserves the trial directory, dumps all raw model output.

# List available task IDs SMALLCODE_DRY_RUN=1 bun scripts/run-baseline.ts | grep "task-id" # Debug a specific task bun scripts/debug-task.ts cap-group-by_1 # Increase turns for harder tasks SMALLCODE_EVAL_MAX_TURNS=10 bun scripts/debug-task.ts cap-group-by_1

Re-run timeout problems from a marathon log

# Reads the log, finds "trial timeout" lines, re-runs those problems # Restart Ollama first: ollama stop vibethinker-3b (reloads on next call) SMALLCODE_HE_K=3 bun scripts/rerun-timeouts.ts /tmp/humaneval-full.log

Check metrics history

bun scripts/show-metrics.ts # Show a transcript bun scripts/show-transcript.ts evals/transcripts/<id>.json

Smoke test the harness

# Quick sanity check — runs E0 tasks, exits 0 if pass@1 >= threshold bun scripts/smoke-e0.ts # Validate capability (E1) fixtures bun scripts/validate-e1.ts # Run bun test suite for the harness itself # bunfig.toml ([test] root = "tests") scopes this to tests/ so the intentionally-RED # seeded-bug fixtures under evals/fixtures/*/tests/ are not discovered (685 pass / 0 fail). # Eval fixtures still grade correctly — the harness runs them in OS-tmp trial dirs # outside the repo tree, which do not inherit this bunfig. bun test

Config and model setup

# Initialize config interactively bun run src/cli/index.ts config init # List available model profiles bun run src/cli/index.ts config models

VibeThinker-3B Modelfile (for top_k=-1)

The OpenAI-compat route silently ignores top_k. To set top_k=-1 (disable top-k sampling as the model was trained), create a Modelfile:

# Modelfile FROM vibethinker-3b PARAMETER top_k -1 PARAMETER num_ctx 8192 # Build and use the custom version ollama create vibethinker-3b-topk -f Modelfile # Then update activeModel in smallcode.config.json

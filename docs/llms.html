Module map

Every important file in src/ and scripts/ with a one-line purpose.

src/agent/

FilePurpose
loop.tsCore agent turn loop — planning, model calls, edit application, tool execution, baseline-relative test-oracle early-stop. Captures a TestBaseline before the loop starts; passes it to every runTieredOracle call. Implements stall detection: STALL_LIMIT=2 consecutive turns with the same failureSignature trigger a redraft (suppresses Recent History, injects rotateStrategy hint); capped at MAX_REDRAFTS=2. Think-only recovery: when a turn truncates mid-reasoning (hasReasoning && answer==="" && truncated!==false) it is recorded as a failed turn and answerNowNext is set so the next turn is built with { answerNow: true } (suppress history, demand immediate action) instead of re-sending the identical prompt that just truncated. Empty-generation infra guard: a turn whose provider returned zero tokens (rawContent.trim()==="" && completionTokens===0 — a wedged/disconnected local backend that emits {"response":"","done":false}) is thrown and recorded with the error tag "infra: empty model generation", distinct from think-only (which HAS reasoning), so the eval can exclude the trial instead of forging a false 0.00. Revert-on-regression: after runTieredOracle returns, if verdict.regressed is true (the edit introduced new failures: newFailures are named, OR total red count exceeded the baseline red count — which catches module-load crashes that print no parseable failure names), the loop calls revertFiles using each ApplyResult.effectivePath (the path actually written, after path-typo rescue) and ApplyResult.originalContent (pre-batch on-disk content) to restore file state precisely. Reverts only on true regression, never when the suite is still-failing because the target isn't fixed yet (progress is kept); "solved" never reverts. On revert the turn's stall signature folds onto the prior turn's so a repeated regress→revert loop still trips the stall/redraft brake. The model receives in the next turn: "⚠ Your edit was REVERTED — it broke tests that were passing before: <names>…". Stores turn.failureSignature, turn.diagnostic, turn.redrafted, turn.answerNow; persists state.stallCount, state.redraftCount, state.lastFailureSignature. Entry point: runLoop(state, statePath, deps, getContext)
planner.tsCalls the model to decompose a task into ≤5 ordered sub-goals (down from 8). Falls back to a single goal if the model returns nothing parseable. Optional preSolveReflection (default off): one extra cheap call to restate the problem + key edge case, fed into goal generation (AlphaCodium-lite). Context summary is sanitised to unique file paths only (no line ranges, capped at 8 files) so the model cannot echo path labels back as goals. Post-parse filtering via isActionableGoal(desc) rejects bare path / path-with-line-range echoes, and isInstructionEcho(desc) rejects goals that parrot the planner's OWN instruction prose (failure mode #7 — markers like "action verb", "sub-goal", "numbered list", literal <action verb>/<specific thing> placeholders, or the copied example verb list "Add, Fix, Implement"); when every goal is filtered out the planner falls back to the raw task. maxGoalsForTask(task, uniqueFileCount) enforces the cap (single-file → 3, multi-file → 5). Exact-duplicate descriptions are deduplicated. The planner system prompt also carries an explicit "NEVER restate these instructions as a goal" guard. All three helpers are exported for testing.
prompt.tsbuildSystemPrompt — delegates to config.promptSet ?? defaultPromptSet(); when the resolved PromptSet.skill is non-empty, appends \n\n## SKILL\n<skill> to the returned string; otherwise output is byte-identical to the no-skill baseline. The discipline-rules toggle relocates to prompt-set.ts. buildTurnPrompt(state, context, opts?) — assembles task, goal, turn number, last 2 turns of history (suppressed on opts.redraft), context chunks, a ## Edit Target directive derived from context.targetFile (small file ≤80 lines → "emit the COMPLETE file" via FILE:; large file >80 lines with a known function → "use PATCH:/FUNCTION: to replace ONLY that one function" — the harness, not the model, picks the format), and a Failure (fix THIS) block rendered from state.turns.at(-1)?.diagnostic when present. When the edit target is an anonymous export default function (…) (synthetic anchor name "default"), the directive prose references the recognizable export default function rather than the opaque "the default function" (which names nothing visible in the source and cost localization confidence on real-repo default exports like mri/klona/dequal); the PATCH FUNCTION: line keeps default as the applier anchor. Format-consistent not-applied feedback: when a SEARCH/REPLACE edit fails to apply (DIFF_EDIT mode, target function ≥ SMALLCODE_DIFF_MIN_FN lines), the retry message instructs the model to re-emit a SEARCH/REPLACE block with the SEARCH text copied byte-for-byte from the shown current file — not "re-emit the complete file", which contradicts the SR directive and causes truncation on large functions. Whole-file and small-function PATCH targets keep their existing not-applied feedback unchanged. When an edit is rejected by the anti-fake-green guard (error contains TEST_FILE_EDIT_REJECTED), the feedback pivots the model to the IMPLEMENTATION file and suppresses the generic "re-emit the file" recovery (which would contradict the rejection and re-show the test). opts.answerNow emits an ANSWER NOW section (skip thinking, act on the first line) and also suppresses history; it takes precedence over redraft. BuildTurnPromptOpts: { redraft?: boolean; strategyHint?: string; answerNow?: boolean }. fitTurnPromptToWindow(state, context, systemPrompt, hardCap, opts?) — pre-flight guard: rebuilds the prompt while shedding the largest non-pinned repo-context chunk each pass until estimateTokens(system) + estimateTokens(prompt) ≤ hardCap or no droppable chunks remain; returns { turnPrompt, estimatedTokens, droppedChunks }. The pinned target file is never shed. History (failing-test output, failed-edit file) is never trimmed. Pure, deterministic, does not mutate the caller's chunk array.
prompt-set.tsPromptSet { system, planner, reflection, skill? } — groups all three agent prompts so a GEPA candidate can carry a mutated variant. The optional skill?: string slot holds a repo-scoped guidance block (mirrors gskill's SKILL.md); when non-empty, buildSystemPrompt appends a ## SKILL section after the base system text. defaultPromptSet(opts?) assembles the canonical set and accepts opts.skill (default undefined). Single source of truth for the prompt strings.
state.tsCreate, load, save (atomic rename), and mutate AgentState. addTurn, advanceGoal, failGoal, isTerminal, currentGoal.
tools.tsTool implementations (read_file, write_file, run_command, run_tests, think, finish) and the executeTool dispatcher. All paths guarded by resolveSafe.
bestofn.tsselectBestCandidate — per-TURN candidate selection: generate N completions in parallel, apply each, run verifier, pick winner by verifierScore, revert losers, apply winner. Off by default.
bestofn-loop.tsRun-level oracle-verified Best-of-N. runBestOfNLoop({n, temperatures, setup, verify, deps}) runs the full agent loop up to N independent attempts at the SAME task — fresh env per attempt (setup), a temperature sweep for diversity — and returns the FIRST attempt whose verify (the sound test oracle) goes green, short-circuiting the rest. defaultTemperatures(n) spreads around 1.0 in [0.7, 1.3] (n=1→[1.0]). Returns {passed, attemptsUsed (≤n), winningAttempt, temperatures, states, modelsUsed, winningModelId}. Per-attempt temp injected via the loop's samplingOverride. Wired into the eval runner via SMALLCODE_BEST_OF_N; converts noisy pass@1 into pass@N(any) with zero selection error. R1 model-escalation: optional models: EscalationRung[] makes the MODEL the per-attempt axis — attempt i retargets state.modelId + deps.profile to models[min(i,len-1)], so a run escalates 3b→7b→14b and only pays for the bigger model on the residual the small model can't solve (resolved on first oracle-green, zero selection error).
escalation.tsR1 ladder builder. buildEscalationLadder({spec, registry, provider}) parses a SMALLCODE_ESCALATION spec (comma-separated model ids, cheapest first) into EscalationRung[] ({id, provider, profile}) for runBestOfNLoop({models}). Rungs share the base provider (local models behind one Ollama endpoint, selected per-request by id → stays offline). Unknown id throws via registry.get (clear config error). Empty/unset spec → undefined (plain temp-swept BoN, no behaviour change).
types.tsTypeScript types: AgentState (+ lastFailureSignature?, stallCount?, redraftCount?, verified? — set true only when the tiered oracle confirmed solved before loop exit), AgentConfig (optional promptSet?: PromptSet for GEPA seam), Goal, TurnRecord (+ failureSignature?, redrafted?, answerNow?, diagnostic?), ToolCall, ToolResult, ToolName.
strategy.tsrotateStrategy(n: number): string — returns the nth strategy hint (0-indexed) from a 3-item cycle: "use a set/dedup explicitly", "step through each provided example by hand first", "handle edge cases (empty, negative, boundary) before the main case". Used by loop.ts to vary the redraft nudge each cycle.
index.tsRe-exports the public agent API including PromptSet, defaultPromptSet.

src/edit/

FilePurpose
parser.tsParse model output into EditBlock[]. Tries PATCH: (function-level) first, then full-file, then search-replace, then JSON. Public: parse(raw).
applier.tsApply EditBlock to disk. Full-file (search='') overwrites the whole file — guarded by truncationReason(original, replace) which rejects likely-truncated re-emits (unbalanced brackets vs a balanced original, >50% line loss, blanking a non-empty file) so a dropped tail never silently corrupts a file. Search-replace tries exact match, then repairBlock fuzzy repair on a miss (records ApplyResult.repair). flattenedPathCandidate(path) + applyBatch redirect a dots-flattened path (src.stats.ts → src/stats.ts) onto the real existing file instead of creating a stray one. patch-function blocks dispatch via a sentinel prefix. Each ApplyResult carries effectivePath (the path actually written, after path-typo rescue) and originalContent (pre-batch on-disk content) — both consumed by loop.ts's revert-on-regression path to restore the correct file even when the model typo'd the path. Anti-fake-green: applyBatch rejects any edit whose path is a test/spec file (isTestFilePath — matches .test./.spec./tests//__tests__/, checked on both the emitted and the flatten-rescued path) with status: "error" + feedback and NO write — the tests are the oracle, so a model editing them to pass can never produce a false "solved". Public: applyBatch, isTestFilePath, truncationReason, flattenedPathCandidate.
patch-function.tsFunction-level edits for large files: parsePatchBlocks, applyPatchBlock (anchors on the function's DEFINITION located by NAME via findDefinitionLines — robust to a small model emitting a drifted signature; the synthetic name "default" anchors on the export default function line; falls back to first-replacement-line match only when name lookup finds nothing; brace-depth scan finds the region end), preserveExport (re-adds a leading export/export default the model dropped — a common slip that un-exports the function and breaks importers), chooseEditFormat (PATCH_LINE_THRESHOLD = 80 lines or >8KB → patch eligible; threshold lowered from 300). The harness — not the model — decides the format deterministically from the pinned target file's size and emits it as the ## Edit Target directive. Fail-safe: ambiguous/not-found anchor → error ApplyResult, no write.
repair.tsFuzzy repair for failed search-replace blocks — strategies: exact → whitespace-normalize → per-line-trim → char-similarity fuzzy (≥0.85). repairBlock(block, content) returns the verbatim recovered search text + strategy/confidence. Wired into applier.ts (no longer dead code): a search/replace that misses exact match retries against the repaired text.
types.tsEditBlock, EditFormat (incl. patch-function), ParseResult, ParseError, ApplyResult, ApplyStatus, ApplyBatchResult, RepairResult.
index.tsRe-exports parse and applyBatch.

src/models/

FilePurpose
registry.tsBuilt-in profiles: vibethinker-3b (reasoning, <think> tags, temp 1.0), the non-reasoning executor coders qwen2.5-coder:3b / qwen2.5-coder:7b (profile id = the Ollama model name; temp 0.7 / top_p 0.8 / top_k 20, num_ctx 8192), and the GEPA reflection model qwen2.5-coder:32b (strong build-time prompt optimizer; contextWindow 32768, num_ctx 16384, max_tokens 8192, temp 0.6, Apache-2.0 — NOT an executor arm, used only by LLMReflectiveMutator via SMALLCODE_GEPA_REFLECT_MODEL). types.tsModelProfile Zod schema and TypeScript type. Fields: id, label, contextWindow, samplingDefaults, reasoningTags, supportsGrammar, supportsJsonSchema, ollamaOptions, notes.

Fixes the HTTP-400 overflow from budgeting off the nominal 64K window.

src/context/

FilePurpose
walker.tsRecursively walk a repo directory, skip node_modules/.git/.smallcode, collect source files by extension.
extractor.tsExtract CodeSymbols (functions, classes, exports, etc.) from source files (TS compiler for TS/JS, regex for other langs). Includes an anonymous export default function (…) as the symbol "default" — common in real libraries (mri, etc.) where the default export IS the main implementation; without it the PATCH target selector could never aim there.
scorer.tsscoreFiles(files, query) ranks files by query-token overlap: exact symbol-name match +15 (EXACT_NAME_WEIGHT — the dominant target signal; one exact ≈ five partials, so a file that DEFINES the named identifier beats a big decoy whose own symbols substring-match a mentioned filename, e.g. token "table" hitting renderTable/renderTableWith), partial symbol-name +3, path-token +2, function/method kind +1. Used to prioritize chunks within the token budget and pick the pin/PATCH target. Measured by scripts/probe-retrieval.ts (deterministic target-correctness across multi-file fixtures: 22/22).
builder.tsBuild a ContextBundle — collect, score, and truncate chunks to fit the token budget (default 8,000 tokens). Target-file pinning: identifies the edit target — the highest query-scored non-test, non-barrel source file (barrel/re-export files doing export * are skipped, they define no implementation) — and emits it as a single PINNED whole-file chunk that is never windowed (always the complete file, never a ±20-line slice) and never shed by fitTurnPromptToWindow. Sets bundle.targetFile = { path, lineCount, format: "full"|"patch", functionName } so the harness can deterministically size-gate the edit format (≤80 lines → full file, >80 → patch one function). Target function (pickTargetFunction): picks the function CONTAINING the bug — a dominant function (covers ≥50% of the file and ≥2× the next) wins query-independently (the common real-repo shape: a default export that IS the implementation), else an exact query name-match, else body-content relevance. Deliberately does NOT trust a weak name-substring match (which once mis-aimed a 9-line helper toVal over a 100-line parser). File tie-break: when several same-named files export the same symbol (a multi-file repo with decoys), the file the repo's TEST files actually import (resolved from their from "…" specifiers) is promoted over non-imported decoys, so the pin lands on the real entry point. Disabled by SMALLCODE_TARGET_PIN=0.
tokens.tsestimateTokens(text) — fast estimate via Math.ceil(text.length / 4).
types.tsContextChunk, ContextBundle, CodeSymbol, FileMap, RepoMap.

src/eval/

FilePurpose
harness.tsTop-level eval harness — load suite, run all tasks, aggregate metrics.
runner.tsrunSuite(suite, opts) — iterate tasks, call runTask, collect EvalRunResult.
task-runner.tsrunTask(task, opts) — create trial env, run agent loop n times (opts.trialsPerTask = sample count), grade each trial, then report pass@k for a SET of k (opts.reportKs, default [1,2,3,5] ∪ {n}) — n is decoupled from the reported k, the core variance fix (the old code only computed pass@k at k=n, collapsing it to a binary). Each pass@k gets a 95% bootstrap CI (passAtKCI) via stats.ts; new opts bootstrapIters, ciSeed. Returns TaskEvalResult with added optional n + passAtKCI. Context comes from the production walkRepo → buildContext path. Reads SMALLCODE_TARGET_PIN (default on; 0 disables target pin + size-gate for a clean pre-A baseline A/B).
stats.tsMeasuring-stick statistics. makeRng(seed) (mulberry32, deterministic) · bootstrapCI(passedFlags, k, opts) — 95% CI for pass@k by resampling the n binary trial outcomes n-with-replacement iters× and recomputing pass@k via computePassAtK (reused, not reimplemented); collapses to a point at c∈{0,n}, flags degenerate at n<2 · passAtKFromFlags · aggregateSuite(taskResults, ks) pooled suite-level pass@k + CI. Pure, seeded → reproducible intervals.
task-loader.tsloadSuite(dir) — read and validate all task JSON files from a suite directory.
trial-env.tscreateTrialEnv(task, fixturesRoot) — copy fixture into a temp dir, return dir path and cleanup fn.
graders/deterministic.tsRun bun test in the trial dir and map exit code + per-file ✓/✗ to pass/fail verdict. Infra-error retry: a transient subprocess failure (InvalidLockfileVersion/EAGAIN/…) that produced ZERO test verdicts is retried up to SMALLCODE_GRADER_RETRIES (default 1) — guarded so a real failing test (a parsed ✗) is NEVER retried. On exhaustion: verdict:"error" with details.infraError=true, so the run excludes that trial from the rate denominator instead of counting it as a model failure.
graders/static.tsRun a configurable static analysis command (e.g. tsc --noEmit) and grade by exit code.
graders/llm.tsCall the model with an evaluation rubric; parse pass/fail from the response.
metrics.tsCompute pass@1, pass@k (unbiased estimator 1−∏(n−c−i)/(n−i)), pass^k, partial scores from trial results. The estimator is correct and unchanged — stats.ts + task-runner.ts now feed it more samples and query it at multiple k.
transcript-store.tsPersist and load Transcript objects to evals/transcripts/.
types.tsEvalTask, EvalSuite, EvalRunResult, TaskEvalResult, TrialResult, GraderConfig, GraderResult, Transcript.

src/improve/ & src/verify/ & src/cli/

FilePurpose
src/improve/session-logger.tsLog agent sessions (status, turns, tokens) to evals/transcripts/ for later task extraction. getFailedSessions(limit?) — filter by outcome === "failed" | "max_turns". getPassedSessions(limit?) — filter by outcome === "done"; used by the skill distiller.
src/improve/skill-distiller.tsOffline template-based skill distillation. distillSkill(sessions, { transcriptStore, maxSessions? }): Promise<string> — reads passing transcripts (cap: maxSessions, default 20), tallies tool sequences, edit formats, common run_command cmds, and stall-recovery events, then emits a concise ## SKILL-style markdown checklist. Deterministic, no model calls. Returns a stub string when there are no passing sessions.
src/improve/task-extractor.tsExtract CandidateTasks from failed session transcripts for promotion to eval suite.
src/improve/promoter.tsWrite candidate tasks as JSON to the eval suite directory.
src/improve/metrics-store.tsAppend MetricsSnapshot to evals/metrics-history.jsonl. Read baseline snapshot for regression gate. MetricsSnapshot (in src/improve/types.ts) gained optional self-describing fields: n, reportKs, perTaskPassAtK, perTaskCI, overallPassAtK, overallCI, thinkOnlyTotal, sampling:{temp,maxTokens} — all optional, so existing readers (regression-gate, show-metrics) are unaffected.
src/improve/regression-gate.tsCompare current eval run against baseline snapshot. Fail if pass@1 regressed or any task moved from pass to fail.
src/improve/ab-compare.tsRun two prompt variants against the same suite and compare pass@1.
src/verify/failure-extract.tsPure-string failure extraction from bun test output (no I/O). extractFirstFailure(output): FailureDiagnostic | null — parses the first (fail)/✗ line, captures Expected/Received values, errorType (via /\b(\w*Error)\b/), and detects module-load crashes (assertionId="<module-load>"). R2 externalize-localization: extractSourceFrame(output) returns the first SOURCE stack frame ({file, line}) — skipping test/spec, node_modules, bun/node internals — so a runtime throw's diagnostic carries sourceFile/sourceLine; a pure value-mismatch trace stops at the test line, so the location is deliberately left undefined there (never surfaces the test line as a false bug location). The loop reads a tight window around it and renders a ## FAILURE LOCATION block (SMALLCODE_LOCALIZE=1, default off). failureSignature(d): string — stable hash; normalises away [Nms] timings, absolute paths, hex addresses, UUIDs, line:col numbers. renderDiagnostic(d): string — compact ≤400-char block for model prompts with Expected/Received.
src/verify/oracle.tsTiered verification oracle. runTieredOracle(repoRoot, opts) — Tier 1: bun test, solved iff the suite is FULLY green (currentRed=0 ∧ ≥1 pass); Tier 2: advisory typecheck fallback when no tests exist. The success tick never shows while any test is red — partial progress reports failing so the loop keeps working, and a repo with unrelated unfixable reds ends in max_turns (honest) rather than a false "verified". opts.baseline shapes both feedback and the regression check: OracleVerdict.regressed is set true when the current turn introduced new failures — either named test IDs appear in newFailures, OR total red count exceeds the baseline red count (count-guard catches module-load crashes that print no parseable failure names). The loop uses regressed to trigger revertFiles; it is NOT set when the suite is still-failing because the target bug isn't fixed yet (that progress is kept). On failing paths sets OracleVerdict.diagnostic via extractFirstFailure. captureTestBaseline(repoRoot) — snapshot failing IDs + red count + loadError before the loop. parseFailingTestIds / parseRedCount — stable parse of (fail) label [Xms] lines and summary counts. R4 validate-before-commit (SMALLCODE_VALIDATE_EDIT, ON by default): hasLoadError(output) detects a suite that failed to LOAD/COMPILE (missing module, parse/transpile error, unhandled error before any test ran). An edit that INTRODUCES a load error (current load-error ∧ baseline had none) is forced to regressed=true regardless of count — fixing the failure mode where a non-loading suite runs fewer tests, the red-count DROPS below baseline, and the count-guard mistakes a broken edit for progress (so the edit was kept and the repo left non-loading). The feedback leads with an actionable BUILD ERROR — your last edit does not compile/load… body; prompt.ts renders a matching ## BUILD ERROR directive (fix the import/syntax, don't invent modules) in place of the assertion block. Set =0 to restore count-only behaviour. Oracle-free static confidence (SMALLCODE_STATIC_CONFIDENCE, default ON): on the no-test "clean" path the verdict carries a confidence grade from confidence.ts so the caller reports what WAS checked instead of a bare "unverified". escalateBrokenClean(verdict) promotes a no-test "clean" verdict whose confidence is broken (the edit does not PARSE) to failing+regressed so the loop's revert + BUILD ERROR prompt + stall all fire — the R4 load-error guard generalized to untested repos (the loop calls it right after runTieredOracle; run.ts surfaces the confidence grade on any unverified end — finish OR max_turns).
src/verify/confidence.tsOracle-free static-confidence ladder. computeStaticConfidence(repoRoot, typecheck) grades a change no test covers: broken (a source file fails Bun-transpiler PARSE — universal, needs no tsconfig) → parses (all source parses) → type-clean (also tsc-clean); lint is an advisory signal only; unknown when nothing checkable. A SAFETY signal, NOT correctness — a wrong operator parses/typechecks/lints clean. renderConfidence(c) → one-line honest summary; the CLI surfaces it on an unverified finish. Measured by scripts/probe-confidence.ts: across fixtures with tests hidden, buggy==solution 5/5 (logic-blind, by design) + structurally-broken downgraded 5/5 (build-break caught).
src/improve/gepa/types.tsGEPA types: Candidate (id, prompts, parentId, generation, scores, meanScore), GepaConfig (taskIds, populationCap, maxGenerations, trialsPerTask), FailedInstance.
src/improve/gepa/pareto-front.tsdominates(a,b,taskIds) — Pareto dominance check. ParetoFront — non-dominated set with population cap and GEPA-weighted selection: each task contributes weight to the front member that maximises it; sample proportional to weight count.
src/improve/gepa/mutator.tsReflectiveMutator interface — mutate(parent, failures). MockMutator — deterministic stub for tests (appends versioned tag, records calls).
src/improve/gepa/reflective-mutator.tsLive reflective mutator. LLMReflectiveMutator implements ReflectiveMutator with an injectable complete(prompt)=>Promise<string> (tests need no network). buildReflectionPrompt(parent, failures, opts) (pure, exported) embeds the parent system verbatim + a capped digest (taskId, outcome, tool-call flag, final-turn rawResponse, ~1500 chars, ≤4 failures) and asks a STRONG model to diagnose the dominant generalizable failure and rewrite system, preserving FILE:/PATCH: + TOOL: protocol and staying task-agnostic; output wrapped in <NEW_SYSTEM>…</NEW_SYSTEM>. ROBUSTNESS: throw / missing block / system <200 chars → returns parent UNCHANGED (no-op) + stderr warning. mutatePlanner (default off) also rewrites planner via <NEW_PLANNER>. makeProviderComplete(cfg) / reflectConfigFromEnv(fallback) — build the complete fn from createProvider + the model registry (no hand-rolled HTTP).
src/improve/gepa/evaluate-adapter.tsevaluateCandidate(cand, deps) — scores a candidate by injecting cand.prompts as AgentConfig.promptSet and calling runTask (or injectable mock) per task. Returns Record<taskId, passAt1>.
src/improve/gepa/engine.tsrunGepa(seed, mutator, evalDeps, cfg, rng?) — evolution loop: select parent → gather failed transcripts → mutate → score → front.add × maxGenerations.
src/improve/gepa/index.tsGEPA barrel re-export (incl. LLMReflectiveMutator, buildReflectionPrompt, makeProviderComplete, reflectConfigFromEnv).
scripts/gepa-smoke.tsSmoke script for the GEPA engine — wires real eval runner + MockMutator against E0–E2, K=1, 3 generations. Created but not run; guarded by import.meta.main.
scripts/gepa-run.tsLive GEPA runner. Loads the edit-reliability suite as the TRAIN set, builds an LLMReflectiveMutator via makeProviderComplete(), runs runGepa (budget via env: SMALLCODE_GEPA_GENERATIONS=3, SMALLCODE_GEPA_TRIALS=5, SMALLCODE_GEPA_POP_CAP=6), prints the final Pareto front + best, and writes the best PromptSet to evals/gepa-best.json. SMALLCODE_GEPA_DRY_RUN=1 → MockMutator + stubbed runTask (no GPU/LLM/network, exits 0). Reflection model via SMALLCODE_GEPA_REFLECT_MODEL (+ optional _BASE_URL/_API_KEY/_MAX_TOKENS/_TIMEOUT — raise the timeout for a slow strong reflector or its call no-ops to the parent); SMALLCODE_GEPA_MUTATE_PLANNER=1 opts into planner rewrites. Guarded by import.meta.main.
scripts/distill-skill.tsCLI that loads the session log + transcript store, filters passing sessions, calls distillSkill, and prints the result to stdout or writes it to --out <path>. Options: --log <path> (default evals/sessions.jsonl), --transcripts <dir> (default evals/transcripts), --max <n> (default 20). Idempotent; no model calls; guarded by import.meta.main.
src/verify/runner.tsRun a configurable checker suite (format, lint, typecheck, test) after edits. Return VerifyResult.

Tool contracts

The model emits tool calls as TOOL: name {"arg": "value"} lines. The harness parses them with a line-level regex and executes each in order. think and finish are control-flow only; others are side-effecting.

read_file
TOOL: read_file {"path": "src/foo.ts"}

Read a file relative to repoRoot. Returns file contents (truncated at 8,000 chars with [truncated] marker). Returns (file not found) if missing. Path must stay within repoRoot (traversal check).

  • path (string, required) — relative path to file
  • Output fed back as tool result in next turn's prompt history
write_file
TOOL: write_file {"path": "src/foo.ts", "content": "..."}

Write content to a file. Creates intermediate directories. If requireApproval: true, throws ApprovalRequiredError before writing. Path traversal check enforced. Prefer the FILE: edit format — it is parsed before tool calls and is the intended write mechanism.

  • path (string, required)
  • content (string, required)
run_command
TOOL: run_command {"cmd": "bun tsc --noEmit"}

Run a shell command in repoRoot. Command basename must be in allowedCommands (default: bun, tsc, biome, git). Timeout: 30 seconds. Output truncated at 4,000 chars. Combined stdout+stderr returned.

  • cmd (string, required) — space-separated argv
  • Returns: { success: exitCode===0, output, durationMs }
run_tests
TOOL: run_tests {}

Run bun test (or bun test <pattern>) in repoRoot. Timeout: 60 seconds. "Solved" means zero new failures relative to a pre-loop baseline AND ≥1 passing test. Pre-existing failures captured in the baseline are excluded, so unrelated failing tests in a real repo never block early-stop. On fresh single-file benchmark repos (empty baseline) the behaviour is identical to the original whole-suite-green check. This is the deterministic pass-oracle: when outcome is "solved", the loop breaks immediately.

  • pattern (string, optional) — test file glob/path
  • Returns: { success: exitCode===0 && failCount===0, output, durationMs }
think
TOOL: think {"content": "reasoning here..."}

No-op control tool. The harness records it but takes no action. Allows the model to emit explicit reasoning steps outside <think> tags. Always returns success.

finish
TOOL: finish {"summary": "implemented add()"}

Signal goal completion. The harness calls advanceGoal — marks the current goal done and increments currentGoalIndex. If all goals are done, status becomes done. Note: if the oracle returns "solved" (zero new failures + ≥1 pass vs baseline), the loop early-stops before checking for finish — the baseline-relative early-stop oracle takes precedence.

  • summary (string, optional) — brief description of what was done

Edit format spec

Primary: full-file

Emit FILE: <path> on its own line, then a fenced code block with the complete corrected file. The search field is empty — the applier overwrites the whole file. This is what the system prompt instructs and what both qwen2.5-coder and VibeThinker-3B reliably emit.

# Full-file format FILE: src/math.ts ```ts export function add(a: number, b: number): number { return a + b; } ``` TOOL: run_tests {} TOOL: finish {"summary": "implemented add()"}

Rules enforced by the parser: path must not be empty, absolute, or contain ... Leading ./ is normalized. Backslashes are converted to forward slashes.

Large files: patch-function

For files past chooseEditFormat's threshold (PATCH_LINE_THRESHOLD = 80 lines or >8KB; lowered from 300), re-emitting the whole file wastes context and risks the small model dropping unchanged lines. The PATCH: format replaces a single function instead. The harness picks this format deterministically from the pinned target file's line count and tells the model via the ## Edit Target directive — the model never self-assesses file size. The anchor is the first line of the replacement body (typically the signature); the applier finds the one matching line, brace-depth-scans to the closing }, and swaps only that region. If the anchor is missing or ambiguous it returns an error ApplyResult (no write) so the loop can fall back to full-file — it never silently mis-applies.

# patch-function format (large files) PATCH: src/foo.ts FUNCTION: parseConfig ```ts export function parseConfig(raw: string): Config { return JSON.parse(raw); } ```

Fallback 1: search-replace

Classic diff-like format. The file path is on the line immediately before the <<<<<<< SEARCH marker (3+ angle brackets, case-insensitive). Small models (incl. VibeThinker-3B) have trouble with this format; use full-file instead.

src/math.ts <<<<<<< SEARCH function add(a, b) { return a; } ======= function add(a: number, b: number): number { return a + b; } >>>>>>> REPLACE

Fallback 2: JSON

JSON object or array with "type": "edit". Extracted from fences or bare JSON spans. Only activated if the raw output contains "type":"edit".

{"type": "edit", "file": "src/math.ts", "search": "return a;", "replace": "return a + b;"}

Parse priority

Priority Format Trigger Notes
1st patch-function PATCH: marker Checked first, but only matches when a PATCH: block is present. The harness gates this format on the pinned target file's size (>80 lines) via the ## Edit Target directive. Anchored, fail-safe.
2nd Full-file FILE: marker Returns immediately if blocks found and valid. Primary/default format.
3rd Search-replace <<< SEARCH / ===== / >>> REPLACE Fallback. VibeThinker-3B unreliable on this format.
4th JSON "type":"edit" anywhere in output Last resort. Also parses from markdown fences.

VibeThinker-3B Modelfile (legacy, for top_k=-1)

qwen2.5-coder models need no Modelfile — their defaults work out of the box. For VibeThinker-3B (origin baseline), the OpenAI-compat route silently ignores top_k. To set top_k=-1 (disable top-k sampling as the model was trained), create a Modelfile:

# Modelfile FROM vibethinker-3b PARAMETER top_k -1 PARAMETER num_ctx 8192 # Build and use the custom version ollama create vibethinker-3b-topk -f Modelfile # Then update activeModel in smallcode.config.json

Module map

Every important file in src/ and scripts/ with a one-line purpose.

src/agent/

FilePurpose
loop.tsCore agent turn loop — planning, model calls, edit application, tool execution, baseline-relative test-oracle early-stop. Captures a TestBaseline before the loop starts; passes it to every runTieredOracle call. Implements stall detection: STALL_LIMIT=2 consecutive turns with the same failureSignature trigger a redraft (suppresses Recent History, injects rotateStrategy hint); capped at MAX_REDRAFTS=2. Stores turn.failureSignature, turn.diagnostic, turn.redrafted; persists state.stallCount, state.redraftCount, state.lastFailureSignature. Entry point: runLoop(state, statePath, deps, getContext)
planner.tsCalls the model to decompose a task into ≤8 ordered sub-goals. Falls back to a single goal if the model returns nothing parseable. Optional preSolveReflection (default off): one extra cheap call to restate the problem + key edge case, fed into goal generation (AlphaCodium-lite).
prompt.tsbuildSystemPrompt — delegates to config.promptSet ?? defaultPromptSet(); the discipline-rules toggle relocates to prompt-set.ts. buildTurnPrompt(state, context, opts?) — assembles task, goal, turn number, last 2 turns of history (suppressed on opts.redraft), context chunks, and a Failure (fix THIS) block rendered from state.turns.at(-1)?.diagnostic when present. BuildTurnPromptOpts: { redraft?: boolean; strategyHint?: string }.
prompt-set.tsPromptSet { system, planner, reflection } — groups all three agent prompts so a GEPA candidate can carry a mutated variant. defaultPromptSet(opts?) assembles the canonical set (discipline-rules toggle honoured). Single source of truth for the prompt strings.
state.tsCreate, load, save (atomic rename), and mutate AgentState. addTurn, advanceGoal, failGoal, isTerminal, currentGoal.
tools.tsTool implementations (read_file, write_file, run_command, run_tests, think, finish) and the executeTool dispatcher. All paths guarded by resolveSafe.
bestofn.tsselectBestCandidate — generate N completions in parallel, apply each, run verifier, pick winner by verifierScore, revert losers, apply winner.
types.tsTypeScript types: AgentState (+ lastFailureSignature?, stallCount?, redraftCount?), AgentConfig (optional promptSet?: PromptSet for GEPA seam), Goal, TurnRecord (+ failureSignature?, redrafted?, diagnostic?), ToolCall, ToolResult, ToolName.
strategy.tsrotateStrategy(n: number): string — returns the nth strategy hint (0-indexed) from a 3-item cycle: "use a set/dedup explicitly", "step through each provided example by hand first", "handle edge cases (empty, negative, boundary) before the main case". Used by loop.ts to vary the redraft nudge each cycle.
index.tsRe-exports the public agent API including PromptSet, defaultPromptSet.

src/edit/

FilePurpose
parser.tsParse model output into EditBlock[]. Tries PATCH: (function-level) first, then full-file, then search-replace, then JSON. Public: parse(raw).
applier.tsApply EditBlock to disk. Full-file (search='') overwrites whole file. Search-replace does exact-match. patch-function blocks dispatch to the patch applier via a sentinel prefix. Public: applyBatch.
patch-function.tsFunction-level edits for large files: parsePatchBlocks, applyPatchBlock (anchor = first replacement line; brace-depth scan finds the region), chooseEditFormat (>300 lines or >8KB → patch eligible). Fail-safe: ambiguous/not-found anchor → error ApplyResult, no write.
repair.tsFuzzy repair for failed search-replace blocks — strategies: exact, fuzzy, whitespace-normalize.
types.tsEditBlock, EditFormat (incl. patch-function), ParseResult, ParseError, ApplyResult, ApplyStatus, ApplyBatchResult, RepairResult.
index.tsRe-exports parse and applyBatch.

src/provider/

FilePurpose
openai-client.tsOpenAI-compatible HTTP client. Sends POST /chat/completions. Forwards ollamaOptions as the options field. Handles streaming.
factory.tscreateProvider(config, registry) — returns an OpenAICompatibleClient, wrapped by maybeWrapWatchdog (on unless SMALLCODE_WATCHDOG=0). Extension point for future backends.
watchdog.tsWatchdogProvider — wraps a Provider, measures real-generation throughput (completionTokens / wall-time). After consecutiveSlow (2) gens below thresholdTps (20 tok/s), runs ollama stop <model> to defrag the KV-cache; next call reloads at full speed. Ignores errored + sub-minTokens (64) gens; never probes (single-slot-safe). Injectable reload/now.
sampler.tsSampling utilities — top-p/top-k post-processing helpers.
types.tsProvider, CompletionRequest, CompletionResponse, ChatMessage, TokenUsage, StreamChunk, ProviderError.

Tool contracts

The model emits tool calls as TOOL: name {"arg": "value"} lines. The harness parses them with a line-level regex and executes each in order. think and finish are control-flow only; others are side-effecting.

read_file

TOOL: read_file {"path": "src/foo.ts"}

Read a file relative to repoRoot. Returns file contents (truncated at 8,000 chars with [truncated] marker). Returns (file not found) if missing. Path must stay within repoRoot (traversal check).

  • path (string, required) — relative path to file
  • Output fed back as tool result in next turn's prompt history

write_file

TOOL: write_file {"path": "src/foo.ts", "content": "..."}

Write content to a file. Creates intermediate directories. If requireApproval: true, throws ApprovalRequiredError before writing. Path traversal check enforced. Prefer the FILE: edit format — it is parsed before tool calls and is the intended write mechanism.

  • path (string, required)
  • content (string, required)

run_command

TOOL: run_command {"cmd": "bun tsc --noEmit"}

Run a shell command in repoRoot. Command basename must be in allowedCommands (default: bun, tsc, biome, git). Timeout: 30 seconds. Output truncated at 4,000 chars. Combined stdout+stderr returned.

  • cmd (string, required) — space-separated argv
  • Returns: { success: exitCode===0, output, durationMs }

run_tests

TOOL: run_tests {}

Run bun test (or bun test <pattern>) in repoRoot. Timeout: 60 seconds. "Solved" means zero new failures relative to a pre-loop baseline AND ≥1 passing test. Pre-existing failures captured in the baseline are excluded, so unrelated failing tests in a real repo never block early-stop. On fresh single-file benchmark repos (empty baseline) the behaviour is identical to the original whole-suite-green check. This is the deterministic pass-oracle: when outcome is "solved", the loop breaks immediately.

  • pattern (string, optional) — test file glob/path
  • Returns: { success: exitCode===0 && failCount===0, output, durationMs }

think

TOOL: think {"content": "reasoning here..."}

No-op control tool. The harness records it but takes no action. Allows the model to emit explicit reasoning steps outside <think> tags. Always returns success.

finish

TOOL: finish {"summary": "implemented add()"}

Signal goal completion. The harness calls advanceGoal — marks the current goal done and increments currentGoalIndex. If all goals are done, status becomes done. Note: if the oracle returns "solved" (zero new failures + ≥1 pass vs baseline), the loop early-stops before checking for finish — the baseline-relative early-stop oracle takes precedence.

  • summary (string, optional) — brief description of what was done

Edit format spec

Primary: full-file

Emit FILE: <path> on its own line, then a fenced code block with the complete corrected file. The search field is empty — the applier overwrites the whole file. This is what the system prompt instructs and what VibeThinker-3B reliably emits.

# Full-file format FILE: src/math.ts ```ts export function add(a: number, b: number): number { return a + b; } ``` TOOL: run_tests {} TOOL: finish {"summary": "implemented add()"}

Rules enforced by the parser: path must not be empty, absolute, or contain ... Leading ./ is normalized. Backslashes are converted to forward slashes.

Large files: patch-function

For files past chooseEditFormat's threshold (>300 lines or >8KB), re-emitting the whole file wastes context and risks the small model dropping unchanged lines. The PATCH: format replaces a single function instead. The anchor is the first line of the replacement body (typically the signature); the applier finds the one matching line, brace-depth-scans to the closing }, and swaps only that region. If the anchor is missing or ambiguous it returns an error ApplyResult (no write) so the loop can fall back to full-file — it never silently mis-applies.

# patch-function format (large files) PATCH: src/foo.ts FUNCTION: parseConfig ```ts export function parseConfig(raw: string): Config { return JSON.parse(raw); } ```

Fallback 1: search-replace

Classic diff-like format. The file path is on the line immediately before the <<<<<<< SEARCH marker (3+ angle brackets, case-insensitive). VibeThinker-3B has trouble with this format; use full-file instead.

src/math.ts <<<<<<< SEARCH function add(a, b) { return a; } ======= function add(a: number, b: number): number { return a + b; } >>>>>>> REPLACE

Fallback 2: JSON

JSON object or array with "type": "edit". Extracted from fences or bare JSON spans. Only activated if the raw output contains "type":"edit".

{"type": "edit", "file": "src/math.ts", "search": "return a;", "replace": "return a + b;"}

Parse priority

PriorityFormatTriggerNotes
1st patch-function PATCH: marker Checked first, but only matches when a PATCH: block is present (large-file opt-in). Anchored, fail-safe.
2nd Full-file FILE: marker Returns immediately if blocks found and valid. Primary/default format.
3rd Search-replace <<< SEARCH / ===== / >>> REPLACE Fallback. VibeThinker-3B unreliable on this format.
4th JSON "type":"edit" anywhere in output Last resort. Also parses from markdown fences.

Config & env vars

smallcode.config.json

Config file searched at smallcode.config.json, then .smallcode.json. Override location with explicit path arg to loadConfig(path).

{ "config": { "provider": { "baseUrl": "http://localhost:11434/v1", // Ollama OpenAI-compat endpoint "apiKey": "none", // unused for local Ollama "timeoutMs": 180000 // 3 min per HTTP call }, "activeModel": "vibethinker-3b", "maxTurns": 15, "bestOfN": 1, "sandbox": { "enabled": true, "requireApproval": false, // false = fully automated "allowedCommands": ["bun", "tsc", "biome", "git"], "networkAccess": false }, "eval": { "suitesDir": "evals/suites", "transcriptsDir": "evals/transcripts", "defaultTrials": 5 } }, "models": [] // optional: custom ModelProfile objects }

How to run

Run the HumanEval-TS benchmark

Fetches problems from the HuggingFace API. Requires Ollama running with VibeThinker-3B.

# First 30 problems, k=3 (reproduces the 0.967 result) SMALLCODE_HE_LIMIT=30 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts # Full set (159 problems, k=3) — expect 8-12 hours SMALLCODE_HE_LIMIT=164 SMALLCODE_HE_K=3 bun scripts/run-humaneval.ts 2>&1 | tee /tmp/humaneval-full.log # Re-run timeout-affected problems with fresh Ollama + 20-min cap bun scripts/rerun-timeouts.ts /tmp/humaneval-full.log

Run the in-house capability suite

# Dry-run: validate all reference solutions pass their graders (no agent needed) SMALLCODE_DRY_RUN=1 bun scripts/run-baseline.ts # Live run: k=3 trials per task with the agent SMALLCODE_EVAL_K=3 bun scripts/run-baseline.ts # Faster: k=1, max 3 turns per task SMALLCODE_EVAL_K=1 SMALLCODE_EVAL_MAX_TURNS=3 bun scripts/run-baseline.ts

Debug a single task

Runs one task end-to-end, preserves the trial directory, dumps all raw model output.

# List available task IDs SMALLCODE_DRY_RUN=1 bun scripts/run-baseline.ts | grep "task-id" # Debug a specific task bun scripts/debug-task.ts cap-group-by_1 # Increase turns for harder tasks SMALLCODE_EVAL_MAX_TURNS=10 bun scripts/debug-task.ts cap-group-by_1

Re-run timeout problems from a marathon log

# Reads the log, finds "trial timeout" lines, re-runs those problems # Restart Ollama first: ollama stop vibethinker-3b (reloads on next call) SMALLCODE_HE_K=3 bun scripts/rerun-timeouts.ts /tmp/humaneval-full.log

Check metrics history

bun scripts/show-metrics.ts # Show a transcript bun scripts/show-transcript.ts evals/transcripts/<id>.json

Smoke test the harness

# Quick sanity check — runs E0 tasks, exits 0 if pass@1 >= threshold bun scripts/smoke-e0.ts # Validate capability (E1) fixtures bun scripts/validate-e1.ts # Run bun test suite for the harness itself bun test

Config and model setup

# Initialize config interactively bun run src/cli/index.ts config init # List available model profiles bun run src/cli/index.ts config models

VibeThinker-3B Modelfile (for top_k=-1)

The OpenAI-compat route silently ignores top_k. To set top_k=-1 (disable top-k sampling as the model was trained), create a Modelfile:

# Modelfile FROM vibethinker-3b PARAMETER top_k -1 PARAMETER num_ctx 8192 # Build and use the custom version ollama create vibethinker-3b-topk -f Modelfile # Then update activeModel in smallcode.config.json

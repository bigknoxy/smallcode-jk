Agent loop flow

The per-session and per-turn cycle. State is persisted to .smallcode/state.json after every turn. The loop terminates when all goals are done, a finish tool call is emitted, or maxTurns is reached.

Module reference

Module Path Purpose Key exports
Config src/config/ Load and validate smallcode.config.json via Zod schemas. Supports env var overrides. SmallcodeConfig, loadConfig, loadConfigFromEnv
Models src/models/ Built-in model profiles (VibeThinker-3B, Qwen2.5-Coder 7B/14B) and a runtime registry for custom profiles. ModelProfile, ModelRegistry, defaultRegistry
Provider src/provider/ OpenAI-compatible HTTP client for chat completions and streaming. Retryable error classification. Provider, CompletionRequest, CompletionResponse, ProviderError
Reasoning src/reasoning/ Strip <think>…</think> traces from raw model output. Log reasoning separately for inspection. ReasoningHandler, ReasoningLogger, ParsedResponse
Context src/context/ Build a token-budgeted ContextBundle from a RepoMap of file symbols and code chunks. RepoMap, FileMap, ContextBundle, ContextChunk
Edit src/edit/ Parse SEARCH/REPLACE blocks and JSON patch edits from model output. Apply exact-match replacements and report diffs. EditBlock, ParseResult, ApplyResult, parse, applyBatch
Agent src/agent/ Core loop, goal management, prompt builders, tool dispatcher, state persistence, and best-of-N sampling. runLoop, AgentState, AgentConfig, TurnRecord, BestOfNResult
Verify src/verify/ Run a configurable checker suite after edits. Feed failures back to the model for self-correction up to maxCorrectionIterations. VerifyResult, CheckResult, SelfCorrectionResult, VerifyConfig
Eval src/eval/ Load task suites, run trials, grade with deterministic tests / static analysis / LLM rubric, compute pass@1 and partial scores. EvalTask, EvalSuite, EvalRunResult, Transcript, GraderResult
Improve src/improve/ Session logging, candidate task promotion, metrics history, regression gating, and A/B prompt comparison. SessionLogger, MetricsStore, RegressionCheckResult, ABResult, runGate

Eval metrics

Metrics computed by the eval harness per task (TaskEvalResult) and aggregated across the whole run (EvalRunResult).

Metric Formula Where What it means
pass@1 P(trial passes | n=1) TaskEvalResult.passAt1 Headline coding metric. Probability that a single sample solves the task. Directly comparable across models and prompt variants.
pass@k P(at least 1 of k trials passes) TaskEvalResult.passAtK[k] Ceiling metric under best-of-N sampling. Shows maximum achievable quality if you can run k inference calls and pick the best.
pass^k (passAllK) P(all k trials pass) TaskEvalResult.passAllK Consistency bar. A model with pass^k ≈ pass@1 is reliable; a large gap signals high variance — increase bestOfN.
partial score graders_passed / graders_total TrialResult.partialScore Partial credit when a task has multiple graders. Useful for tracking directional progress on tasks that aren't passing end-to-end yet.
overall pass@1 tasks_passed / tasks_total EvalRunResult.overallPassAt1 Suite-level headline. Used by the regression gate and A/B comparison as the single summary number.
n_turns count of TurnRecord entries TrialMetrics.nTurns Efficiency signal. Fewer turns = simpler task decomposition required. High turn counts indicate complex or ambiguous tasks.
n_total_tokens prompt_tokens + completion_tokens TrialMetrics.nTotalTokens Cost proxy for local inference. Useful for comparing prompt strategies that solve the same task.

Regression gate logic

The gate compares the current run's overallPassAt1 against a stored baseline snapshot. It fails if currentPassAt1 < threshold or if any previously-passing task now fails (regressedTasks.length > 0). Exit code 1 is used to block CI merges.

Security controls

smallcode is designed for local use with untrusted model output. The following controls prevent a misbehaving model from escaping the repo sandbox or executing arbitrary commands.

Control Where enforced What it prevents
Command allowlist src/agent/tools.ts, src/verify/sandbox.ts The agent and the verification runner both check the command basename against sandbox.allowedCommands before execution. Commands not on the list are rejected with an error returned to the model — not silently dropped. Default allowlist: bun, tsc, biome, git.
Path traversal check src/agent/loop.ts (safeResolve), src/verify/sandbox.ts (checkFilePath) All file paths are resolved to absolute form and checked that they start with repoRoot + path.sep. Any path that escapes the repo root is rejected before a read or write is attempted. This applies to both agent-initiated edits and verifier-run test commands.
Approval gate src/agent/tools.ts, AgentConfig.requireApproval When sandbox.requireApproval is true (the default), destructive file writes prompt the user for confirmation before proceeding. Disabling this should only be done in fully automated CI environments.
Network access flag src/config/types.ts, sandbox enforcement sandbox.networkAccess: false (default) signals that agent-run commands should not make outbound network requests. Used as a policy signal to sandbox wrappers and is logged for audit.
State file permissions src/agent/state.ts, src/eval/transcript-store.ts Session state (.smallcode/state.json) and eval transcripts are written to project-local directories, not global paths. No credentials, API keys, or system paths are written into state files. The config loader reads apiKey from config but never serializes it into state or transcripts.
Read-only reasoning log src/reasoning/logger.ts Reasoning traces extracted from <think> tags are logged separately and never fed back into the prompt context. This prevents a prompt-injection attack where the model embeds instructions inside its own reasoning output.
